The test suite validates user flows and is ideal for regression test use; when new releases are published, we can have some confidence and evidence that the system is still working, including authentication with valid and invalid credentials, shopping cart operations including adding and removing items, the multi-step checkout process with required field validation, and user navigation between pages. The tests can identify authentication failures, missing server-side validation, cart data persistence issues, session management problems, and UI interaction failures.

These tests were chosen in order to tackle potential login authentication risks the product might have. Also, testing the end-to-end flow of how a real user would use the website reveals critical findings that could prevent users from making purchases, and eventually the business would be impacted.
